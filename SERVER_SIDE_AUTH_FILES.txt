## Rollout Safety Checklist

Before touching client files (Phase 4):
- ✅ Backend server starts without errors
- ✅ All auth endpoints respond correctly (Postman)
- ✅ GitHub proxy works for GET requests
- ✅ Unit tests pass for server/

During client refactoring (Phase 4):
- ⚠️  App WILL break temporarily (localStorage → API)
- ⚠️  Keep old code in comments for quick rollback
- ⚠️  Test each file change individually

After client refactoring (Phase 5):
- ✅ Integration tests pass
- ✅ E2E tests pass
- ✅ Manual smoke test: login → boards → logout

## Quickstart for Implementer

1. **Setup backend (1-2 days):**
   ```bash
   cd apps/flowboard
   npm install express iron-session cookie-parser @types/express @types/cookie-parser tsx
   mkdir -p server/{middleware,routes}
   touch server/{index,types}.ts server/middleware/{session,auth}.ts server/routes/{auth,github-proxy}.ts
   ```

2. **Implement backend (1-2 days):**
   - Copy session config template from IPD §4.3
   - Implement auth routes following IPD §5.2
   - Implement proxy following IPD §5.3
   - Test with Postman

3. **Update client (1-2 days):**
   - Start with sessionStore.ts (fetch API instead of localStorage)
   - Update LoginView.tsx (call /api/auth/login)
   - Update App.tsx (call /api/auth/user on mount)
   - Update AppShell.tsx (call /api/auth/logout)
   - Update client.ts (call /api/github/* instead of api.github.com)

4. **Verify & document (1 day):**
   - Run all tests
   - Update README
   - Write ADR-009

**Total: 4-7 days for experienced developer**

## MVP Shortcut (Fastest Path to Working Demo)

**Day 1:** Backend skeleton
- server/index.ts (minimal Express + iron-session)

**Day 4:** Testing & polish
- Fix bugs
- Add basic tests
- Update README

**Result:** Working prototype in 4 days, then iterate to production-ready in remaining time.
